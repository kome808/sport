# 2626/1/28 ANTIGRAVITY 系統開發規格書：疲勞監測參數與警示邏輯

ANTIGRAVITY 系統疲勞監測與警示規格書

**1. 核心指標：訓練負荷比 (ACWR) - 採用 EWMA 模型**

**目的：** 評估運動員「短期疲勞」與「長期體能」的平衡狀態。此指標為系統預測受傷風險的核心數據。 **科學依據：** EWMA 模型比傳統滾動平均 (RA) 更能靈敏地反映近期的負荷變化，對傷害預測的敏感度更高。

**A. 計算公式 (EWMA Algorithm)**

系統需每日針對每位選手計算兩個數值：`Acute Load (EWMA_7)` 與 `Chronic Load (EWMA_28)`。

1. **定義衰減係數 (Lambda,** *λ***)：**

◦ *λa* (急性係數，N=7天): 2/(7+1)=0.25

◦ *λc* (慢性係數，N=28天): 2/(28+1)≈0.069

2. **每日 EWMA 計算邏輯：**

◦ **今日急性負荷 (***EWMAacute***)** = (今日*sRPE*×0.25)+((1−0.25)×昨日*EWMAacute*)

◦ **今日慢性負荷 (***EWMAchronic***)** = (今日*sRPE*×0.069)+((1−0.069)×昨日*EWMAchronic*)

◦ *注意：若「昨日」無數據（系統第一天），則 EWMAyesterday*=今日*sRPE。*

3. **ACWR 比值計算：**ACWR=今日慢性負荷 (*EWMAchronic*)今日急性負荷 (*EWMAacute*)

**B. ACWR 警示區間 (Traffic Light Logic)**

根據文獻，設定以下閾值：

- **🟢 綠燈 (最佳區間)：0.80 ～ 1.30**

◦ 意義：訓練負荷適中，受傷風險最低 (Sweet Spot)。

- **🟡 黃燈 (注意區間)：1.31 ～ 1.49** 或 **< 0.80**

◦ 意義 (>1.3)：負荷增加中，需密切觀察。

◦ 意義 (<0.8)：訓練量不足 (Undertraining)，體能可能衰退，未來受傷風險亦會增加。

- **🔴 紅燈 (警戒區間)：1.50 ～ 1.99**

◦ 意義：短期負荷過大，進入危險區 (Danger Zone)。

- **⚠️ 紫燈/深紅 (極高風險)：≥ 2.0**

◦ 意義：極端負荷尖峰 (Spike)。研究顯示 ACWR > 2.0 時，非接觸性受傷風險可能增加 5-7 倍。

**2. 生理指標：晨間心跳 (RHR)**

**目的：** 反映自律神經系統狀態與恢復能力。 **科學依據：** 靜止心率升高是過度訓練的早期指標，升高 5-10 bpm 具有顯著臨床意義。

**A. 計算邏輯**

1. **基準線 (Baseline)：** 計算該選手「過去 7 天」的 RHR 移動平均值 (7-day Rolling Average)。

2. **差異值 (Delta)：** `今日 RHR` - `基準線`。

**B. RHR 警示區間**

- **🟢 綠燈 (正常)：** 差異值 ≤ 4 bpm (或低於基準線)。
- **🟡 黃燈 (注意)：** 差異值在 **+5 ～ +9 bpm** 之間。

◦ 意義：身體處於壓力狀態，建議觀察主觀感受。

- **🔴 紅燈 (警戒)：** 差異值 ≥ **10 bpm**。

◦ 意義：強烈的疲勞、生病或過度訓練訊號，建議當日休息。

- -------------------------------------------------------------------------------

**3. 主觀指標：身心狀態 (Wellness)**

**目的：** 捕捉客觀數據無法量化的壓力（睡眠、心情、生活壓力）。 **科學依據：** 主觀感受往往比客觀數據更早預測疾病或表現下滑，建議觀察與個人平均值的偏差 (Z-score 或標準差)。

**A. 計算邏輯**

1. **總分計算：** 加總問卷各項分數（如：疲勞+睡眠+痠痛+壓力+心情）。假設使用 1-10 分制，5題滿分50分。

2. **個人基準線：** 計算過去 28 天的「平均總分」與「標準差 (SD)」。

3. **Z-score (標準分數)：** (今日總分−28天平均總分)/28天標準差。

◦ *若系統初期無法計算 Z-score，可改用「低於平均值百分比」作為替代方案。*

**B. Wellness 警示區間 (以 Z-score 為例)**

- **🟢 綠燈 (正常)：** Z-score > -1 (分數在平均值附近或更高)。
- **🟡 黃燈 (注意)：** Z-score 介於 **1 到 -2** 之間 (分數比平常低 1 個標準差)。
- **🔴 紅燈 (警戒)：** Z-score **< -2** (分數比平常低 2 個標準差以上)。

◦ 意義：身心狀態顯著惡化。

- -------------------------------------------------------------------------------

**4. 負荷總量指標：今日訓練負荷 (sRPE)**

**目的：** 監控絕對負荷量的波動，避免單週增加過快。 **科學依據：** 每週負荷增加量超過 10-15% 或絕對值增加過多，會顯著提升受傷風險。

**A. 計算邏輯**

1. **sRPE (AU)：** `RPE (0-10)` × `訓練時間 (min)`。

2. **週負荷變化率 (Week-to-Week Change)：**上週累計負荷本週累計負荷−上週累計負荷×100%

**B. sRPE 警示區間**

- **🟢 綠燈 (安全)：** 週變化率 < 10%。
- **🟡 黃燈 (注意)：** 週變化率 10% ～ 15%。
- **🔴 紅燈 (警戒)：** 週變化率 **> 15%** (或單週總量增加 > 1000 AU)。
- -------------------------------------------------------------------------------

**5. 高風險名單篩選演算法 (Risk Flagging Algorithm)**

系統將於每日數據輸入後執行此邏輯，只要滿足 **任意一項** 紅燈條件，即將選手列入「高風險關注名單」。

**判定邏輯 (IF-THEN 規則)：**

1. **IF** `ACWR` > 1.5 **THEN** Status = **RED** (標記：負荷過度 spikes)

2. **OR IF** `RHR_Delta` ≥ 10 bpm **THEN** Status = **RED** (標記：生理疲勞/潛在疾病)

3. **OR IF** `Wellness_Z-score` < -2 **THEN** Status = **RED** (標記：身心狀況極差)

4. **OR IF** `Weekly_Load_Change` > 15% **THEN** Status = **RED** (標記：訓練量暴增)

5. **ELSE IF** (任一指標為黃燈) **THEN** Status = **YELLOW** (標記：需觀察)

6. **ELSE** Status = **GREEN** (狀態良好)

**ANTIGRAVITY 系統儀表板顯示建議**

- **視覺化：** 針對每位選手，除了顯示數值外，務必使用顏色（綠/黃/紅）直觀呈現上述四項指標的狀態。
- **趨勢圖：** ACWR 需提供折線圖，並在背景畫出 0.8-1.3 的綠色區間帶，讓教練一眼看出選手是否脫離安全區。
- **EWMA 權重說明：** 在系統說明頁面備註：「本系統採用 EWMA 模型計算負荷，該模型給予近期訓練較高權重，能更精準捕捉急性疲勞累積。」

**⚠️ 教練與家長的使用指南：**

1. **紅燈停，綠燈行：** 當任一指標出現**紅色警報**時，請務必重視。這不代表選手「偷懶」，而是身體機能已經亮紅燈，硬練只會導致受傷或生病。

2. **看趨勢，不看單點：** 不要因為某一天的數據不好就過度驚慌，重點是觀察**連續 3-7 天的趨勢**。例如 RHR 連續三天都高於平均 5 下，比單一天高 5 下更值得注意。

3. **多指標綜合判斷：**

◦ 若 **ACWR > 1.5** (練太多) + **Wellness 變差** (覺得累) = **高受傷風險，必須減量**。

◦ 若 **RHR 升高** + **Wellness 變差** = **可能生病或過度訓練，建議休息**。
